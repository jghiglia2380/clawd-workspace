# Impact Report Template
## Admin-Ready Summary for Instructors

### Purpose
A shareable one-pager that instructors can give to their administration to demonstrate the impact of PFL Academy in their classroom. This is the "make me look good" document.

---

## Design Principles

1. **Professional** — Looks like something from a real organization
2. **Data-Forward** — Numbers are prominent and compelling
3. **Scannable** — Admin can get the gist in 30 seconds
4. **Attributable** — Clearly credits the instructor
5. **Contextual** — Connects to broader educational goals

## Content Sections

### Header Block
| Field | Source | Required |
|-------|--------|----------|
| Report Title | Static: "CLASSROOM IMPACT REPORT" | Yes |
| Instructor Name | Profile | Yes |
| School Name | Profile | Yes |
| District Name | Profile | Yes (if available) |
| Course Name | Profile | Yes |
| Reporting Period | Selected by instructor | Yes |

### Key Metrics Block
| Metric | Description | Format |
|--------|-------------|--------|
| Students Enrolled | Total in reporting period | Number |
| Business Plans | Completed plans | Number |
| Projected Revenue | Sum of all student projections | Currency |
| Completion Rate | % completing curriculum | Percentage |

### Curriculum Coverage
Checkboxes showing which major areas were covered:
- [ ] Financial Literacy Fundamentals
- [ ] Business Planning & Strategy
- [ ] Market Research & Analysis
- [ ] Revenue Modeling & Projections
- [ ] Pitch Development & Presentation

Plus: Standards alignment statement (customizable per state)

### Student Outcomes
Bullet list of notable achievements:
- Assessment scores
- Tangible outputs (plans, presentations)
- External recognition
- Real-world applications

### Featured Student Work
One highlighted project with:
- Business name
- Student name (first + last initial)
- Brief description
- Key metric (revenue projection)
- Notable outcome (if any)

### Instructor Quote
Optional personal statement from the instructor

### Footer
- PFL Academy branding
- Website
- Report generation date

## Customization Options

### Instructor Can Choose:
- Reporting period (semester, year, custom date range)
- Which student work to feature
- Personal quote (or use default)
- Whether to include Page 2 details
- Privacy level (download only vs. shareable link)

### System Generates:
- All metrics (auto-calculated)
- Charts and visualizations
- Comparison to platform averages
- Standards alignment text (based on state)

## Technical Notes for Seb

### Data Requirements
All metrics should be calculable from existing data:
- Student enrollment counts
- Completion timestamps
- Assessment scores
- Business plan submissions
- Business plan financial fields

### PDF Generation
Options:
1. **Server-side** — Node library (PDFKit, Puppeteer)
2. **Client-side** — jsPDF or html2pdf
3. **Third-party** — DocRaptor, PDFShift

Recommendation: Puppeteer for pixel-perfect output from HTML template

### API Endpoints
```
GET /api/impact-report/preview?period=fall-2024
POST /api/impact-report/generate — Returns PDF blob
POST /api/impact-report/share — Creates shareable link, returns URL
```

### Template System
- Store report templates as HTML/CSS
- Inject data into templates
- Render to PDF

### Shareable Links
- Generate unique slug: `/reports/[instructor-id]/[report-id]`
- Optional password protection
- Expiration option
- View count tracking
